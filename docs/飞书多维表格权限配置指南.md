# 飞书多维表格权限配置指南

## 问题现象

当系统提示以下错误时，说明权限配置不正确：

- ❌ **HTTP 403 Forbidden** - 获取字段配置失败
- ❌ **错误码 1254302 (RolePermNotAllow)** - 查询记录失败
- ❌ 选择人员栏位是空的（通讯录权限不足）
- ❌ 选择事项栏位是空的（表格权限不足）

---

## 解决方案（按顺序操作）

### 步骤一：确认飞书应用权限配置

#### 1. 登录飞书开放平台

访问：https://open.feishu.cn/app/

#### 2. 进入你的应用 → "权限管理"

确认以下权限已添加并**发布**：

| 权限范围           | 权限代码                        | 说明                   | 状态    |
| ------------------ | ------------------------------- | ---------------------- | ------- |
| 多维表格           | `bitable:app`                   | 访问多维表格应用       | ✅ 必须 |
| 通讯录（基本信息） | `contact:contact.base:readonly` | 获取通讯录用户基本信息 | ✅ 必须 |

⚠️ **注意**：权限名称必须完全正确！

**常见错误**：

- ❌ `contact:user.base:readonly` - 旧版本权限名称，已失效
- ✅ `contact:contact.base:readonly` - 正确的权限名称

**备选权限**（如果找不到上面的权限，可以使用以下任一）：

- `contact:contact:readonly`
- `contact:contact:access_as_app`
- `contact:department.organize:readonly`

#### 3. 检查权限是否已发布

- 权限状态必须是：**已发布** ✅
- 如果显示"待发布"或"审核中"，需要：
  1. 点击"创建版本"
  2. 填写版本说明
  3. 提交审核
  4. 审核通过后点击"申请发布"

---

### 步骤二：多维表格授权给应用（关键！）

这是最容易被忽略的步骤，**必须操作**！

⚠️ **重要提示**：只有以下方法有效，其他入口（如"协作者"、"高级权限"）无法搜索到应用！

#### ✅ 唯一有效的操作步骤

1. **打开你的多维表格**

2. **点击右上角 "..." → "更多"**

3. **选择 "添加文档应用"**

   - ⚠️ 注意：必须是"添加文档应用"，不是"协作者"或"高级权限"

4. **搜索并添加你的应用**

   - 在搜索框输入你的应用名称："人力占用记录系统"
   - 或者输入你的 `app_id`: `cli_a9d5f559aab81cc9`
   - 在搜索结果中找到你的应用

5. **设置权限级别**

   - 选择权限：**可编辑** 或 **管理者**
   - 点击"确定"或"添加"

6. **验证应用已添加**
   - 返回多维表格
   - 可以在表格的"已添加应用"列表中看到你的应用
   - 状态显示为"已启用"或"已添加"

---

#### ❌ 以下方法不可行（已验证）

**不要尝试以下入口，它们无法搜索到应用：**

- ❌ **"协作者"入口** - 搜索不到应用
- ❌ **"高级权限"入口** - 搜索不到应用
- ❌ **"添加协作者"入口** - 搜索不到应用

**唯一有效的入口是：右上角 "..." → "更多" → "添加文档应用"**

---

### 步骤三：验证配置

#### 3.1 刷新你的应用页面

访问：http://localhost:3000/workload

按 `Ctrl + F5` 强制刷新页面

#### 3.2 检查是否成功

✅ **成功标志**：

- 选择人员下拉框：能看到企业成员列表
- 选择事项下拉框：能看到事项选项
- 选择日期和人员后：能看到已有记录

❌ **如果还是失败**：

- 查看浏览器开发者工具（F12）→ Console
- 查看服务器日志中的详细错误信息
- 继续下面的排查步骤

---

## 详细排查步骤

### 问题 1: 获取用户列表失败（HTTP 400）

**可能原因**：

- Contact API 权限未添加
- 权限未发布

**解决方法**：

1. 检查飞书开放平台 → 权限管理
2. 确认 `contact:user.base:readonly` 权限已发布
3. 等待 1-2 分钟让权限生效

---

### 问题 2: 获取字段配置失败（HTTP 403）

**可能原因**：

- 表格未授权给应用
- 应用没有表格的访问权限

**解决方法**：

1. **重新检查步骤二**：确保在表格设置中添加了应用
2. 确认应用权限级别至少是"可编辑"
3. 尝试移除应用后重新添加

---

### 问题 3: 查询记录失败（错误码 1254302）

**错误信息**：`RolePermNotAllow`

**含义**：应用没有表格的角色权限

**解决方法**：

#### 方法 A: 检查表格高级设置

1. 打开表格 → 右上角"..." → "高级设置"
2. 查看"协作者"列表
3. 确认你的应用在列表中
4. 确认权限级别：**至少是"编辑者"**

#### 方法 B: 使用 API 添加权限（高级）

如果界面操作无效，可以使用飞书 API 添加权限：

```bash
# 使用tenant_access_token调用
POST https://open.feishu.cn/open-apis/bitable/v1/apps/{app_token}/members
{
  "member_type": "app",
  "member_id": "你的app_id",
  "perm": "full_access"
}
```

---

### 问题 4: 环境变量配置错误

**检查 `.env` 文件**：

```env
# 多维表格配置
FEISHU_APP_TOKEN=PaBpbkEWxxq2rcCWHxnfd    # ✅ 从日志看已正确
FEISHU_TABLE_ID=tblHxxwuaLkzq                 # ✅ 从日志看已正确

# 应用配置
FEISHU_APP_ID=cli_a9d5f5xx81cc9               # ✅ 需要检查是否正确
FEISHU_APP_SECRET=xxxxxxxxxxxxxxxxxx             # ❓ 需要检查是否正确
```

**验证方法**：

1. 登录飞书开放平台
2. 进入你的应用 → "凭证与基础信息"
3. 对比 `App ID` 和 `App Secret` 是否一致

---

## 测试 API 调用（排查工具）

### 使用飞书 API Explorer 测试

访问：https://open.feishu.cn/api-explorer/

#### 测试 1: 获取用户列表

```
GET /open-apis/contact/v3/users?user_id_type=user_id&page_size=10
Authorization: Bearer {tenant_access_token}
```

**预期结果**：返回用户列表

#### 测试 2: 获取表格字段

```
GET /open-apis/bitable/v1/apps/{app_token}/tables/{table_id}/fields
Authorization: Bearer {tenant_access_token}
```

**预期结果**：返回字段配置

#### 测试 3: 查询表格记录

```
POST /open-apis/bitable/v1/apps/{app_token}/tables/{table_id}/records/search
Authorization: Bearer {tenant_access_token}
Body: {"page_size": 10}
```

**预期结果**：返回记录列表

---

## 权限生效时间

- **应用权限发布**：立即生效
- **表格协作者添加**：通常 1-2 分钟
- **权限缓存清除**：最多 5 分钟

**建议**：配置完成后等待 2-3 分钟再测试

## 常见错误代码参考

| 错误码   | 含义             | 解决方法                 |
| -------- | ---------------- | ------------------------ |
| 400      | 请求参数错误     | 检查 API 参数格式        |
| 403      | 权限不足         | 添加应用权限             |
| 1254302  | 表格角色权限不足 | 在表格中添加应用为协作者 |
| 1254045  | 字段类型不匹配   | 检查写入数据的字段类型   |
| 99991664 | 应用未获取权限   | 在开放平台添加权限并发布 |

## 快速检查清单

配置完成后，逐项检查：

- [ ] 飞书开放平台已添加 `bitable:app` 权限
- [ ] 飞书开放平台已添加 `contact:user.base:readonly` 权限
- [ ] 飞书开放平台已添加 `contact:user.employee_id:readonly` 权限
- [ ] 权限版本已创建并发布（状态显示"已发布"）

- [ ] `.env` 文件中的 `FEISHU_APP_ID` 和 `FEISHU_APP_SECRET` 正确
- [ ] `.env` 文件中的 `FEISHU_APP_TOKEN` 和 `FEISHU_TABLE_ID` 正确
